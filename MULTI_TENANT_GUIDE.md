# Rawaa ERP - نسخة Multi-Tenant (عملاء متعددين)

## الحل: كل عميل له حسابه الخاص وبياناته المستقلة

### للعميل:
1. **نسخة منفصلة على جهازه:**
   - كل عميل يحصل على نسخة محمولة (.exe أو RawaaERP.bat)
   - البيانات محفوظة محليا على جهازه فقط
   - لا تشارك بيانات مع عملاء آخرين

2. **أو نسخة واحدة على السحابة مع multi-tenant:**
   - عميل واحد يسجل دخول → يرى بياناته فقط
   - عميل ثاني يسجل دخول → يرى بياناته (مختلفة تماما)

### كيفية الاستخدام:

#### الطريقة 1: نسخة محمولة لكل عميل (الأفضل)
```bash
# لكل عميل نسخة منفصلة على جهازه
npm run build
pkg . --targets win-x64 --output "Rawaa-Client-1.exe"
npm run build
pkg . --targets win-x64 --output "Rawaa-Client-2.exe"
```

كل .exe مستقل تماما - بيانات منفصلة.

#### الطريقة 2: نسخة واحدة مع multi-tenant
```bash
# Backend واحد في السحابة
# كل عميل يسجل دخول من اسم مستخدم مختلف
# البيانات تنقسم حسب الـ tenant ID
```

إرسال الطلب مع tenant ID:
```javascript
fetch('/api/products', {
  headers: {
    'x-tenant-id': 'client-1' // أو 'client-2'
  }
})
```

### المميزات:
✅ كل عميل بيانات مستقلة تماما
✅ آمن - لا تسرب بيانات بين العملاء
✅ بسيط - بدون تعقيدات قاعدة بيانات
✅ موثوق - كل جهاز مستقل

### الخلاصة:
**الطريقة الأسهل:** نسخة محمولة لكل عميل على جهازه الخاص
